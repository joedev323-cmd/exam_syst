this is a simple exam system that enables users(invigilators ) to mark the attendance 
of the people who are doing the exam without relying solely on the headcount

head count can be erroneous at times;

proposed system counts the students as they enter the examroom
this will act as an assistant not replacing the invigilators

a  backup in case the list gets lost


functional requirements
1) to find a way to capture the data 
2) use the exam card qr as a refference
3) be able to set the expected exam at a given time
4) find a way to use unit code as a factor
5) make it real time
6) have a dashboard with the analytics(current exam + attendance )
7) to beable to generate a report for a given time period
8) analyse the qr and pick relevant info

the proposed language to be used is java (can change to py after research).
